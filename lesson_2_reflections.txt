How is the staging area different from the working directory and the repository? What value do you think it offers?
the staging area contains files you want to work on as part of a logical change, the working directory contains all the files in the project. the repository contains commits

How can you use the staging area to make sure you have one commit per logical change?
you can use the staging area to make sure you have one commit per logical by staging only files that are part of the logical change
